WAI Initiative

Standards and implementation aids are developed by the W3C Web Accessibility Initiative (WAI) to assist in understanding and implementing accessibility.

4 Tenets of Web Accessibility

WCAG has been developed with four primary accessibility guiding concepts in mind. The term POUR stands for Perceivable, Operable, Understood, and Robust, and it refers to these four concepts.

  1. Perceivable by at least one sense
  2. Operable, using keyboard, mouse, voice...
  3. Understandable, it should make sense to the users
  4. Robust, content should use web standards compatible with different browsers, currently and in the foreseeable future
WCAG-EM

WCAG-EM outlines a process for assessing websites and incorporates guidelines to support best practices and assessors.

Importance of Using Semantic Tags

Using semantic tags in HTML has several benefits. Search engines and other user devices can better understand the significance and context of web pages thanks to the semantic HTML elements. Semantically constructed pages are far simpler to read. It is more easily accessible. Better user experience is provided by it.

Using ARIA in Web Pages

Accessible Rich Internet Applications (ARIA) is a set of roles and attributes that define ways to make web content and web applications (especially those developed with JavaScript) more accessible to people with disabilities.
